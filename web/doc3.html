Part 3 - Web App Usage Intro

This part contains general tip for using the web app.

Access the web app at http://bible.madharasan.com. Most of the website is self explanatory. Some helpful tips are available below.

You will need a modern web browser with HTML 5 and JavaScript support.

General Life-cycle of a Chapter Review

  1. Choose an available chapter from chapter list page. Note that once you have chosen a chapter you cannot edit/choose other chapters until you have completed the chosen one. This is because other users will not be able to edit your chapter unless you have submitted it for review or abandon the chapter.
  2. Edit it/Format it/Proofread it
  3. Submit it for review
  4. Other users review it
  5. Then admin will merge it into the main database and upload it to github

The chapters are color coded as follows:

  • Chapters available for editing are colored in Orange.
  • Chapters waiting to be reviewed are colored in Blue.
  • completed chapters are colored in Green.
  • Your Chosen chapter is colored in Teal.
  • Locked chapters are colored in Gray. Some chapters are locked because of technical difficulties in representing them in as a mark-up.

This color combination is used all through out the site. Once you have chosen a chapter to edit, other available chapters will be disabled for you. Your chosen chapter will be disabled for others too.

Markup editor

Code editor page has three tabs.

  • Edit tab to edit the mark-up
  • Preview tab to see live preview of the mark-up in HTML
  • And difference tab to see the difference between the source text and the edited text
WARNING: Viewing difference for very large chapters might crash some old browsers or make it unresponsive. Save your code locally before viewing difference.

The edit tab contains these buttons:

  • 4. The last button, ie., the abandon button, will free up the chapter for others to edit. Your edits will be lost. Please note that even partial edits can be submitted for review.
  • 3. Discard changes button will restore the original untouched mark-up in the text box so you can start over again.
  • 2. Save locally button will save a local copy of your edits in your browser. This feature can be used to restore edits in between sessions, in case of a browser crash. Please note that this does not automatically save, you have to manually do it. This feature requires a browses that supports local storage. And most importantly if you abandon, logout, clear your browser cache, or *have logged in on a different machine* your edits saved locally on a machine will be lost.
  • 1. Expand Abbreviation button is used to expand most commonly used abbreviation. Please see Documentation Part 4 - Punctuation for more details on what is being expanded in here.
Note that the submit for review button is available at the bottom of the preview tab.
You can submit even partial edits.

Rest of the website is explanatory. If you are stuck at something, please let me know.

Oh, one more thing, your user profile picture is taken from Gravatar. If you don't know what it means, no problem as it is not needed for using this website.
