Part 2 - Mark-up

This contains the syntax of the mark-up code used to represent a bible chapter.

The bible texts are auto-coded and presented for editing in the following mark-up. This is to help in the reviewing process. This autogenerated mark-up can be edited in a web interface that is provided. The usage of this interface is discussed in the next part.

General Structure

Mark-up is subdivided into units of:

  • Titles
  • Verses (Verses also contain sub-markup for quotes, poems etc.,)
  • Footnotes and Cross References
  • Para breaks
Each of these section start and end with a tilde symbol ~ .
Line breaks are very important in this mark-up. 
~
Mark up text goes here
~

Comments

You can place comments in your code for reviewers to note. Use # to denote that it is a comment. It has to be in its own section surrounded by tilde symbol as shown here: 

				
~
# Hello this is a comment.
Some more comments I guess...
~

Para Breaks

Para Breaks are denoted by {br} or by inline element {p}. The easiest and the fastest way to insert Para Breaks are by placing {p} at the desired location. But {br} can also be placed next to the verse after which a para break should occour. Note the tilde symbol in the code, they are not part of the verse but a seperate section on their own.This method is highly discouraged. 

				~
v/1
தொடக்கத்தில் கடவுள் விண்ணுலகையும், மண்ணுலகையும் படைத்தபொழுது,{p}
~
{br}
~

Mark-up Verse

A verse Mark-up has two parts (lines). First line contains the Verse number and the second contains the actual verse:

				~
v/1
தொடக்கத்தில் கடவுள் விண்ணுலகையும், மண்ணுலகையும் படைத்தபொழுது,
~
  • Here v/1 - Here the letter V denotes that the following line is a verse. The number that follows after the slash denotes that it is Verse number 1.
  • Continuous verses can be represented as v/7-19 or as v/14,18. The first verse in the continuous verses will have the complete verse text and rest of the verses will have same as above in the place of the verse. 
    				~
v/7-8
அதற்கு எஸ்தர், “என் விண்ணப்பமும் வேண்டுகோளும் இதுவே: மன்னரின் கண்களில் எனக்குத் தயவு கிடைத்தால் மன்னரின் பார்வையில் இது நலமெனத் தோன்றினால், மன்னரும் ஆமானும் நான் நாளைய தினமும் உங்களுக்காய் ஆயத்தம் செய்யும் விருந்திற்கு வருகை தருமாறு வேண்டுகிறேன். அவ்வமயம் மன்னரின் வார்த்தையின்படி என் விண்ணப்பத்தைத் தெரிவிப்பேன்” என்று பதிலிறுத்தாள்.
~
v/8
Same as above
~
  • In the printed version of the bible, for sub verses, english alphabet and tamil alphabets are used interchangebly. Let us use only english alphabets. That makes it uniform and easier for database retrival.
  • The verse can contain other mark-ups for poem and indention etc., which are discussed later in this page.

Mark-up Header

Similar to a verse mark-up, header mark-up too has two parts. First line conains the Verse number after which the header occurs and the second contains the actual verse: 

				~
t/0
கடவுள் உலகைப் படைத்தல்
~

Here t/0 - Here the letter t denotes that the following line is a header(title). The number that follows after the slash denotes that this header should come *after the verse 0* (that is before verse 1).

Case 1:
If header occurs within a verse (See part 1 for example) use {t} to denote the location. 
					~
v/4
இவையே விண்ணுலக, மண்ணுலகப் படைப்பின் தோற்ற முறைமையாம்.{t}ஆண்டவராகிய கடவுள் மண்ணுலகையும், விண்ணுலகையும் உருவாக்கிய பொழுது,
~
t/4
ஆண் - பெண் படைப்பு
~
  • Here the verse will be split at the location of {t} and the header text will be inserted there.
  • Note that if the header occurs within the verse, the mark-up for the header is placed *after the verse*.
Case two:
  • If there are multiple levels of header available, then use {h} to denote their location. Multiple level does not mean, multiple headers, but headers which acts as a title for multiple chapters. These headers divide the entire book into units.
  • If a single level header is written in two lines, then use {br} to indicate the break point.
Example code with both of the above point in a single header (மத்தேயு 1): 
					~
t/0
இயேசுவின்  பிறப்பும்  குழந்தைப் பருவமும்{h}இயேசுவின் மூதாதையர் பட்டியல்{br}(லூக் 3:23 - 38)
~
will be generated as:
<h1>இயேசுவின் பிறப்பும் குழந்தைப் பருவமும்</h1>
<h2>இயேசுவின் மூதாதையர் பட்டியல் <br/>(லூக் 3:23 - 38) </h2>
And rendered as:

இயேசுவின் பிறப்பும் குழந்தைப் பருவமும்

இயேசுவின் மூதாதையர் பட்டியல்
(லூக் 3:23 - 38)

Note that the second header is split into two lines, and is of the same style.

Indent and Outdent

Indent and outdent are represented using {i} , {/i} , {o} and {/o} respectively. Example 

				~
t/45
மரியாவின் பாடல்
~
v/46
{o}அதைக் கேட்ட மரியா பின்வருமாறு கூறினார்:{/o}
~

Poems

Similar to indents above, poems are marked with {p1} , {/p1} and {p2} , {/p2} for poem style 1 and 2 respectively. You can also use {q1} , {/q1} and {q2} , {/q2} . But note that it is split into multiple lines. These lines will be automatically tagged and displayed. 

				~
v/47
{p1}“ஆண்டவரை எனது உள்ளம் 
 போற்றிப் பெருமைப் படுத்துகின்றது. 
 என் மீட்பராம் கடவுளை நினைத்து 
 எனது மனம் பேருவகை கொள்கின்றது.{/p1}
~

Style 3 or `poem para` style is achieved by simply using {p1} and {/p1} in a single line (without line breaks). 

				~
v/1
{p1}கலத்திலிருக்கும் நறுமணத் தைலம் முழுவதையும்  செத்தக்கள் முடைநாற்றம் வீசும்படி செய்துவிடும். அதுபோல சிறிய மதிகேடும் மேன்மையான ஞானத்தைக் கெடுத்து விடும்..{/p1}
~

Another example in style 1 with multiple "first line out", திருப்பாடல்கள் 119:98 which we saw in Introduction part of this documentation will be coded as: 

~
v/98
{p1}என் எதிரிகளைவிட என்னை{br}ஞானியாக்கியது உமது கட்டளை;
ஏனெனில், என்றென்றும் அது{br}என்னோடு உள்ளது.{/p1}
~

Note the placement of {br} tags to get the desired effect.

It is always advisable to stick to the formatting of the print version. But because of the limitations of the print version, which web version does not have, it is not always necessary to do so. Use your own judgement to decide on how to format texts, especially poems.

When formatting a poem, consider the aesthetic of the verse when it is rendered. There are some nouns (names of people and places) which are hyphenated by default. But there are other long words which are hyphenated because of space constraints. *Do not hyphenate such words*. For example in Ps. 124:1 the word இஸ்ரயேல் is broken into two parts because of space constraints: Do not break such words. In short use your personal judgement on when to or not to break.

Expand Abbreviation button in the web app will help you to code poems easily. Please read next and the final chapters to know more about its functionality.

Footnotes and Cross references

Mark-up for cross reference is placed at the end of the code. Cross references mark-up is placed first followed by footnotes. Note that they are arranged one per line as follows:

~
c/3 2 கொரி 4:6
c/6-8 2 பேது 3:5
c/26 1 கொரி 11:7
c/27 மத் 19:4; மாற் 10:6
c/27-28 தொநூ 5:1-2
f/27 ‘ஆதாம்’ என்பது எபிரேய பாடம்
~

Note that for some, you might want to add coustom footnot reference: 1:1d-k எஸ் (கி) 10:3 a-f. This can be achieved by using square brackets: 

~
c/1 [1:1d-k] எஸ் (கி) 10:3 a-f.
~

Eventhough this will not render properly in the preview tab, this will be correctly saved in the database.
